# 随心记 — 产品定义文档

> 当前版本：v0.5.1 | 更新日期：2026-04-17

---

## 一、产品概述

### 1.1 产品定位

「随心记」是一款个人记账微信小程序，核心解决三个痛点：

1. **手动录入效率低** — 通过 AI 视觉识别账单截图，批量导入消费记录
2. **花完不知道钱去哪了** — 通过「日常/额外」双层分类 + 标签体系，清晰区分日常基线支出和额外专项支出
3. **支付渠道与账单来源难对应** — 自动标记支付渠道（花呗/信用卡/微信支付/银行卡），避免重复计算

### 1.2 目标用户

- 主要用户：个人记账
- 协作场景：家人查看消费情况，未来支持双人独立记账+合并查看

### 1.3 核心目标

**快速知道：日常花多少 + 额外花在哪**

## 二、核心概念

### 2.1 分类 + 标签 双层体系

一笔账单同时拥有「分类」和「标签」两个维度：

| 维度 | 数量 | 用途 | 示例 |
|------|------|------|------|
| 分类 | 1个 | 判断是日常还是额外 | 餐饮、交通、购物 |
| 标签 | 多个 | 交叉归属 | 福建旅游、出差 |

**场景**：旅游期间的餐饮消费 → 分类=餐饮，标签=福建旅游，级别=额外

### 2.2 日常 vs 额外

| 类型 | 说明 | 示例 |
|------|------|------|
| 日常 | 每月稳定支出，波动不大 | 餐饮、交通、通讯 |
| 额外 | 非日常大额支出，需单独追踪 | 旅游、装修、医疗 |

### 2.3 支付渠道

| 渠道 | 对应来源 |
|------|---------|
| 花呗 | 支付宝 |
| 信用卡 | 银行APP |
| 微信支付 | 微信 |
| 银行卡 | 银行APP |

### 2.4 项目类型

| 类型 | 匹配方式 | 适用场景 | 示例 |
|------|---------|---------|------|
| 📅 时间段 | 按起止日期 | 有明确时间范围 | 旅游、出差 |
| 🏷️ 关键词 | 按商户关键词 | 按类型归集 | 3D打印机、装修 |

## 三、功能架构

### 3.1 页面结构

```
随心记
├── 首页（月度总览）
│   ├── 日常/额外支出概览 + 占比图
│   ├── 额外项目列表
│   └── 最近账单（点击跳转编辑）
├── 记账（两个模式）
│   ├── 拍照记账
│   │   ├── 上传账单截图 → AI解析
│   │   ├── 解析结果确认列表
│   │   ├── 单条调整（分类/渠道/级别/标签）
│   │   └── 策略批量调整
│   └── 手动记账
├── 报表
│   ├── 总览（分类占比柱状图）
│   ├── TOP10 支出排行
│   ├── 趋势（多月数据积累后展示）
│   └── 月度/年度对比
├── 项目（额外专项管理）
│   ├── 活跃项目列表（显示类型标签）
│   ├── 新建项目（时间段/关键词）
│   └── 项目详情（关联账单+预算）
├── 设置
│   ├── AI模型配置（多提供商）
│   ├── 日常基线预算
│   ├── 密码保护
│   ├── 分类管理
│   └── 策略模板
└── 账单编辑（独立页面）
    ├── 修改所有字段
    ├── 移出项目
    └── 删除
```

### 3.2 AI 识别流程

```
用户上传账单截图（最多5张）
    │
    ▼
AI 视觉模型解析（用户自配 API Key）
    │  Prompt: 解析账单截图，返回结构化 JSON
    │  返回: [{date, merchant, amount, category, confidence}, ...]
    │
    ▼
解析结果确认列表
    ├── ✅ 高置信度 → 自动确认
    ├── ⚠️ 中置信度 → 待确认
    └── ❌ 低置信度 → 待确认
    │
    ▼
用户调整
    ├── 单条调整：修改分类/渠道/级别/标签
    └── 策略调整：按条件批量修改
    │   ├── 默认模板：旅游专项、装修专项、大额标记、出差期间
    │   ├── 条件：时间范围/分类/商户关键词/金额区间
    │   ├── 操作：添加标签/归入项目/标记级别
    │   └── 预览匹配结果 → 确认执行
    │
    ▼
确认入账
```

### 3.3 策略调整

策略是批量修改账单属性的规则，支持预览和模板：

**条件**：
- 日期范围（如 3.3~3.10）
- 分类（如 餐饮）
- 商户关键词（如 酒店）
- 金额区间（如 > 500）

**操作**：
- 添加标签（如 福建旅游）
- 归入项目（如 福建旅游专项）
- 标记级别（日常 → 额外）

**默认模板**：

| 模板名 | 项目类型 | 条件 | 操作 |
|--------|---------|------|------|
| 旅游专项 | 时间段 | 日期范围（待填） | 添加标签+归入项目+标记额外 |
| 装修专项 | 关键词 | 商户关键词（待填） | 添加标签+归入项目+标记额外 |
| 大额标记 | — | 金额 > 500 | 标记额外 |
| 出差期间 | 时间段 | 日期范围（待填） | 添加标签 |

## 五、技术方案

### 5.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 微信小程序 + WeUI | 手机端最佳体验 |
| 图表 | 简易 CSS 柱状图 | 后续可接入 ECharts 小程序版 |
| 数据存储 | wx.setStorageSync（本地存储） | 无需服务器，即开即用 |
| AI 解析 | 视觉大模型 API | 用户自配 Key，支持多提供商 |
| 版本管理 | Git + Tag | 每版本打 tag 记录 release note |

### 5.2 AI 模型支持

| 提供商 | 默认模型 | API 地址 |
|--------|---------|---------|
| OpenAI | gpt-4o | https://api.openai.com |
| Claude | claude-sonnet-4-20250514 | https://api.anthropic.com |
| 通义千问 | qwen-vl-max | https://dashscope.aliyuncs.com/compatible-mode |
| DeepSeek | deepseek-chat | https://api.deepseek.com |

用户可在设置页自定义 API 地址（支持代理/中转）。

### 5.3 AI 解析 Prompt

```
你是一个账单解析助手。请解析这张账单截图，提取每一条消费记录。

返回JSON数组，每条记录包含：
- date: 日期（格式 MM/DD）
- merchant: 商户名称
- amount: 金额（数字，正数表示支出）
- category: 建议分类（从以下选择：餐饮/交通/购物/居住/娱乐/通讯/医疗/教育/其他）
- confidence: 识别置信度（high/medium/low）

仅返回JSON数组，不要其他内容。
```

### 5.4 数据存储方案

当前使用本地存储（wx.setStorageSync），数据以 JSON 字符串存储在以下 key 中：

| Key | 内容 |
|-----|------|
| bills | 所有账单 |
| projects | 所有项目 |
| strategies | 所有策略 |
| user_config | 用户配置 |

**后续升级路径**：切换到微信云开发（云数据库 + 云函数），实现云端同步和多设备访问。
